This document describes what is implemented, what is skeleton code, and how the

================================================================================
                              OVERVIEW
================================================================================

The card module handles CRUD (Create, Read, Update, Delete) operations for
flashcards within decks. Each card belongs to exactly one deck, and users can
only access cards in decks they own.

Architecture:
  - cards.py (Routes Layer) - HTTP endpoints, request/response handling
  - card_service.py (Service Layer) - Business logic, database operations

================================================================================
                         WHAT IS FULLY IMPLEMENTED
================================================================================

1. ROUTE ENDPOINTS (cards.py)
-----------------------------

All routes require JWT authentication via the @jwt_required() decorator.

| Method | Endpoint                        | Description                    |
|--------|---------------------------------|--------------------------------|
| POST   | /decks/<deck_id>/card           | Create a new card              |
| GET    | /decks/<deck_id>/cards/<card_id>| Get a single card              |
| POST   | /decks/<deck_id>/cards/<card_id>| Update a card (edit)           |
| DELETE | /decks/<deck_id>/cards/<card_id>| Delete a card                  |
| GET    | /decks/<deck_id>/cards          | List all cards (paginated)     |

Note: The edit endpoint uses POST (not PUT) per the design document.

Required fields for card creation:
  - word (string) - The word in the source language
  - translation (string) - The translated word
  - word_roman (string) - Romanization/pronunciation guide

Optional fields:
  - definition, word_example, trans_example, trans_roman, image


2. SERVICE LAYER METHODS (card_service.py)
------------------------------------------

CRUD Operations (Fully Working):

  create_card(user_id, card_data)
    - Verifies user owns the target deck
    - Inserts card into database with all provided fields
    - Returns the created card with its generated c_id
    - Triggers TTS generation and image download (skeleton - see below)

  get_card(user_id, card_id, deck_id=None)
    - Retrieves a single card by ID
    - Verifies ownership via deck -> user relationship
    - Returns all 20 database columns including FSRS scheduling fields
    - Returns None if card not found or unauthorized

  update_card(user_id, card_id, update_data, deck_id=None)
    - Updates any subset of card fields
    - Triggers TTS regeneration if word/translation changes (skeleton)
    - Triggers image re-download if image URL changes (skeleton)
    - Returns updated card

  delete_card(user_id, card_id, deck_id=None)
    - Removes card from database
    - Cleans up associated MinIO objects (skeleton)
    - Returns True on success

Query Methods (Fully Working):

  get_cards_for_deck(user_id, deck_id, page=1, per_page=50)
    - Paginated retrieval of all cards in a deck
    - Returns: { cards: [...], pagination: { page, per_page, total, total_pages } }

  get_cards_for_review(user_id, deck_id, limit=20, include_new=True)
    - Gets cards where due_date <= now OR due_date is NULL (new cards)
    - Orders by: new cards first, then by earliest due_date
    - Used by the FSRS spaced repetition algorithm

  get_decks_with_due_cards(user_id)
    - Returns list of decks that have cards needing review
    - Includes due_count and total_cards for each deck

  get_recent_decks(user_id, limit=3)
    - Returns most recently reviewed decks
    - Used for "Continue Learning" feature on home screen

Helper Methods:

  _verify_deck_ownership(user_id, deck_id)
    - Checks if user owns the specified deck
    - Returns True/False

  _get_deck_info(deck_id)
    - Retrieves deck metadata (name, word_lang, trans_lang)
    - Used to determine language for TTS generation

  _is_url(value)
    - Static method to check if a string is an HTTP/HTTPS URL
    - Used to determine if image field contains a URL to download


3. DATABASE SCHEMA COVERAGE
---------------------------

The Cards table has 20 columns. All are handled:

Content Fields (user-provided):
  - c_id (auto-generated primary key)
  - d_id (foreign key to Decks)
  - word, translation, definition
  - word_example, trans_example
  - word_roman, trans_roman
  - image (stores MinIO object ID, not URL)

Media Fields (auto-generated):
  - word_audio (MinIO object ID for TTS audio)
  - trans_audio (MinIO object ID for TTS audio)

FSRS Scheduling Fields (managed by FSRS algorithm):
  - learning_state (0=New, 1=Learning, 2=Review, 3=Relearning)
  - step (current step in learning/relearning sequence)
  - difficulty (0.0 to 1.0, card difficulty rating)
  - stability (days until 90% recall probability)
  - due_date (when card should be reviewed next)
  - last_review (timestamp of most recent review)
  - successful_reps (count of successful reviews)
  - fail_count (count of failed reviews)


4. ERROR HANDLING
-----------------

Custom exception classes are defined and used consistently:

  CardNotFoundError - Card doesn't exist or user unauthorized
  DeckNotFoundError - Deck doesn't exist or user unauthorized  
  UnauthorizedError - User doesn't have permission
  DatabaseError - PostgreSQL operation failed
  MinIOError - Object storage operation failed (defined but not yet used)

-----------------------

MinIO is an S3-compatible object storage system used to store:
  - Card images (downloaded from URLs provided by users)
  - TTS audio files (generated for word and translation)

Skeleton Methods:

  _init_minio()
    Status: SKELETON
    Purpose: Initialize MinIO client connection on service startup
    Current: Does nothing (pass statement)
    Needs: MinIO Python SDK, environment variables for credentials
    Env vars needed: MINIO_ENDPOINT, MINIO_ACCESS_KEY, MINIO_SECRET_KEY, MINIO_BUCKET

  _download_and_store_image(image_url, card_id)
    Status: SKELETON
    Purpose: Download image from URL, upload to MinIO, return object ID
    Current: Downloads image (working), but MinIO upload is commented out
    Returns: Placeholder object ID like "images/1/uuid.jpg"
    Needs: Uncomment MinIO put_object call, handle errors

  _delete_from_minio(object_id)
    Status: SKELETON  
    Purpose: Delete an object from MinIO bucket
    Current: Prints debug message, returns True
    Needs: Uncomment MinIO remove_object call


2. TEXT-TO-SPEECH (TTS) GENERATION
----------------------------------

  _generate_and_store_tts(text, language, card_id, field_type)
    Status: SKELETON
    Purpose: Generate speech audio for word/translation, store in MinIO
    Current: Prints debug message, returns placeholder object ID
    Returns: Placeholder like "audio/1/word_uuid.wav"
    Needs: Integration with TTSService (from services/tts_service.py)

When fully implemented, TTS will:
  - Auto-generate audio when cards are created
  - Regenerate audio when word or translation is updated
